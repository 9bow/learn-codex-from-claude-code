## Codex 재탄생: 2025년 이전과 이후

"Codex"라는 이름은 사실 두 번 사용되었습니다. 2021년 등장한 코드 완성 API (GitHub Copilot의 기반)가 첫 번째였고, 2025년부터는 완전히 새로운 **대화형 AI 코딩 에이전트** 로 재탄생했습니다.

이 가이드에서 다루는 Codex는 모두 2025년 이후의 새로운 Codex입니다.

:::caution[이름 혼동 주의 — 2021 Codex vs 현재 Codex]
**2021년 Codex 코드 모델** (구 제품)과 **현재 Codex** (2025년~)는 완전히 다른 제품입니다.

| 구분 | 2021년 Codex | 현재 Codex (2025~) |
|------|-------------|------------------|
| 성격 | 코드 완성 API | 대화형 AI 코딩 에이전트 |
| 역할 | GitHub Copilot 동력원 | 터미널 / 클라우드 / 데스크톱 에이전트 |
| 현황 | 2023년 deprecated | 현재 활발히 개발 중 |
| 오픈소스 | 아님 | CLI: github.com/openai/codex |

OpenAI가 같은 이름을 재사용했을 뿐, 아키텍처·목적·API 모두 다릅니다.
:::

## 주요 이정표

| 시기 | 이벤트 |
|------|--------|
| 2021-08 | **구 Codex 코드 모델** 출시. GitHub Copilot 동력원. 2023년 deprecated |
| 2025-04-16 | **Codex CLI** 오픈소스 출시 (github.com/openai/codex). 터미널 에이전트로 재탄생 |
| 2025-05 | **Codex 클라우드 에이전트** 출시. ChatGPT 내 백그라운드 SWE 에이전트 |
| 2026-05 기준 | 공식 모델 문서에 `gpt-5.5`, `gpt-5.4`, `gpt-5.4-mini`, `gpt-5.3-codex`, `gpt-5.3-codex-spark`, `gpt-5.2` 라인업이 정리됨 |
| 2026-05 기준 | Codex app은 macOS와 Windows를 지원하며, worktree, automations, Git 기능을 내장 |
| 2026-05 기준 | Agent Skills는 Codex CLI, IDE extension, Codex app에서 사용 가능 |

:::note[참고]
모델 접근 범위와 사용 한도는 자주 바뀔 수 있으므로, 이 챕터의 세부 모델 정보는 항상 공식 Models와 Pricing 문서를 기준으로 확인하세요.
:::

## 현재 모델 라인업 (2026-05)

```toml
# ~/.codex/config.toml 에서 기본 모델 설정
model = "gpt-5.5"
```

| 모델 | 플래그 | 특징 | 비고 |
|------|--------|------|------|
| `gpt-5.5` | `-m gpt-5.5` | 최신, 최고 성능 | ChatGPT 인증만, API 키 미지원 |
| `gpt-5.4` | `-m gpt-5.4` | 권장 폴백, agentic 작업 최적화 | API 키 지원 |
| `gpt-5.4-mini` | `-m gpt-5.4-mini` | 빠르고 효율적 | 가벼운 코딩 작업과 subagent에 적합 |
| `gpt-5.3-codex` | `-m gpt-5.3-codex` | 코딩 특화 | API 키 지원 |
| `gpt-5.3-codex-spark` | `-m gpt-5.3-codex-spark` | 즉각 반복, 빠른 피드백 루프 | ChatGPT Pro only |
| `gpt-5.2` | `-m gpt-5.2` | 이전 세대 | 레거시 호환용 |

:::tip[팁]
API 키로 사용할 때 권장 모델은 **`gpt-5.4`** 입니다. `gpt-5.5` 는 성능이 더 좋지만 현재 API 키로는 접근할 수 없습니다.
:::

## Agent Skills 표준화의 의미

2026년 3월 **agentskills.io** 에 공개된 Agent Skills 표준은 Anthropic이 Claude Code에서 먼저 도입한 Skills 개념을 오픈 표준으로 정식화한 것입니다. Codex가 공식 채택하면서 두 도구 사이의 Skills 호환성이 생겼습니다.

<ConceptCard title="Agent Skills 표준이 중요한 이유">
- **도구 독립적**: Claude Code용 skill과 Codex용 skill이 같은 `SKILL.md` 형식을 공유
- **커뮤니티 생태계**: `github.com/openai/skills` 에 큐레이트된 공식 skill 카탈로그 존재
- **점진적 공개**: skill 디렉토리 스캔 시 이름+설명만 먼저 로드 (컨텍스트 절약)
- **암시적 호출**: description 매칭으로 자동 호출, 명시적으로는 `$skill-name` 입력
</ConceptCard>

## Desktop App이 가져온 변화

Desktop App 출시 전까지 Codex는 Claude Code와 마찬가지로 터미널 중심 도구였습니다. Desktop App은 다음을 바꾸었습니다.

- **워크트리 기본 지원**: 에이전트마다 격리된 git 워크트리에서 작업. 로컬 브랜치 오염 없음
- **병렬 에이전트 시각화**: 여러 작업의 진행 상황을 한 화면에서 관리
- **Automations**: 일일 릴리스 브리프, CI 실패 요약, 이슈 분류 등 스케줄 작업 자동화

Claude Code 사용자 관점에서 Desktop App은 oh-my-claudecode 같은 멀티 에이전트 오케스트레이터를 기본 내장한 것과 유사한 경험을 제공합니다.

## 핵심 정리

- 2021년 구 Codex 코드 모델(Copilot 동력원)과 현재 Codex는 **이름만 같은 완전히 다른 제품** 입니다
- **Codex CLI** 는 오픈소스 CLI이며, **Codex Cloud** 는 ChatGPT 기반 백그라운드 작업을 제공합니다
- **GPT-5.5** 는 공식 Models 문서 기준 Codex의 권장 최신 모델이며, ChatGPT 인증으로 사용할 수 있고 API 키 인증은 지원하지 않습니다
- **Codex app** 은 macOS와 Windows에서 worktree, automations, Git 기능을 제공합니다
- **Desktop App** 은 워크트리 격리와 멀티 에이전트 관리를 기본 제공합니다
