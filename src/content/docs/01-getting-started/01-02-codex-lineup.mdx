## 왜 라인업을 먼저 알아야 하는가

Claude Code는 기본적으로 **하나의 CLI 도구** 입니다. VS Code 확장과 JetBrains 플러그인이 있지만, 핵심은 `claude` 명령어 하나입니다.

Codex는 다릅니다. **다섯 가지 형태** 로 존재하며, 각각 다른 사용 상황에 최적화되어 있습니다. 어떤 형태를 선택하느냐에 따라 설정 방법과 기능이 달라지므로, 전환 전에 전체 그림을 파악하는 것이 중요합니다.

## Codex 5가지 제품 형태

| 형태 | 실행 방법 | 주 용도 | 공유 설정 |
|------|-----------|---------|-----------|
| **Codex CLI** | `codex` (터미널) | 로컬 개발, CI/CD, 자동화 스크립트 | `~/.codex/config.toml` |
| **Codex IDE 확장** | VS Code 계열 / JetBrains 패널 | 에디터 내 AI 지원 | `~/.codex/config.toml` 공유 |
| **Codex Desktop App** | macOS / Windows 앱 | 멀티 에이전트 관리, diff 검토 | CLI 세션·설정 그대로 활용 |
| **Codex Cloud** | chatgpt.com/codex / CLI | 백그라운드 클라우드 작업, GitHub·Slack 통합 | 별도 클라우드 environment |
| **ChatGPT 웹·iOS** | chatgpt.com / 앱 | 일상 대화 + 코드 작업 혼합 | ChatGPT 계정 연동 |

:::note[참고]
CLI와 IDE 확장은 **같은 `~/.codex/config.toml`** 을 읽습니다. CLI에서 모델을 바꾸면 IDE 확장에도 즉시 반영됩니다.
:::

## 각 형태 상세

### Codex CLI

Rust로 작성된 오픈소스 CLI입니다. `github.com/openai/codex` 에서 소스를 볼 수 있습니다.

```bash
# 설치
npm install -g @openai/codex

# 기본 실행
codex "테스트를 추가해 줘"

# 비대화형 실행 (CI 등)
codex exec "린트 오류 모두 수정해"
```

Claude Code의 `claude -p "..."` 에 대응하는 비대화형 모드가 `codex exec` 입니다.

### Codex IDE 확장

VS Code, Cursor, Windsurf 같은 VS Code 계열 IDE와 JetBrains IDE 통합을 지원합니다. CLI와 IDE 확장은 같은 Codex 인증 캐시와 설정을 사용합니다.

:::tip[팁]
Profiles 기능(`[profiles.<name>]`)은 현재 **Experimental** 상태이며 IDE 확장에서는 미지원입니다. CLI에서만 `codex --profile <name>` 으로 사용 가능합니다.
:::

### Codex Desktop App

2026년 3월 macOS 출시 후 같은 달 4일 Windows로 확장된 GUI 앱입니다. 단순한 에디터 확장과 달리 **멀티 에이전트 명령 센터** 역할을 합니다.

주요 특징:
- 여러 에이전트를 동시에 실행하고 각각의 진행 상황을 관리
- **워크트리 기본 지원**: 각 에이전트가 격리된 코드 복사본에서 작업하므로 로컬 git에 영향 없음
- diff 검토 + 코멘트 + 에디터 즉시 열기
- Skills 생성·관리 전용 UI
- Automations: 스케줄 기반 백그라운드 자동 작업

### Codex Cloud

로컬 환경 없이 클라우드에서 백그라운드로 작업을 실행합니다. GitHub 이슈나 Slack 메시지를 트리거로 Codex 에이전트를 자동 실행하는 통합이 포함됩니다.

### ChatGPT 웹·iOS Codex

ChatGPT 인터페이스에서 Codex 기능을 사용합니다. `gpt-5.5` 모델은 현재 ChatGPT 인증을 통해서만 접근 가능하며 API 키로는 사용할 수 없습니다.

<ConceptCard title="Claude Code와 제품 구조 비교">

**Claude Code**:
- `claude` CLI 하나가 핵심
- VS Code 확장 (`anthropic.claude-code`), JetBrains 플러그인은 CLI 래퍼
- 설정: `~/.claude/settings.json`

**Codex**:
- CLI / IDE / Desktop App / Cloud / ChatGPT 5가지 형태
- CLI + IDE 확장은 `~/.codex/config.toml` 공유
- Desktop App은 CLI 세션·설정을 그대로 활용
- Cloud와 ChatGPT는 별도 인증 체계

## 어떤 형태를 선택해야 하는가

| 상황 | 권장 형태 |
|------|-----------|
| 기존 Claude Code처럼 터미널 중심 개발 | **Codex CLI** |
| 에디터를 벗어나지 않고 AI 지원 | **IDE 확장** |
| 여러 에이전트를 동시에 관리하고 싶다 | **Desktop App** |
| GitHub Actions, CI 파이프라인 통합 | **Codex Cloud** |
| 가볍게 코드 관련 질문·수정 | **ChatGPT 웹·iOS** |

Claude Code에서 전환하는 경우 대부분은 **Codex CLI** 부터 시작하는 것이 가장 빠른 적응 경로입니다.

## 핵심 정리

- Codex는 CLI, IDE 확장, Desktop App, Cloud, ChatGPT **5가지 형태** 로 존재합니다
- **CLI와 IDE 확장** 은 `~/.codex/config.toml` 을 공유합니다
- **Desktop App** 은 멀티 에이전트 관리와 워크트리 격리를 기본 지원합니다
- **Codex Cloud** 는 GitHub·Slack 통합 백그라운드 작업에 적합합니다
- Claude Code 전환자에게는 **Codex CLI** 가 가장 친숙한 출발점입니다
- `gpt-5.5` 는 현재 ChatGPT 인증 전용이며 API 키로는 사용할 수 없습니다
