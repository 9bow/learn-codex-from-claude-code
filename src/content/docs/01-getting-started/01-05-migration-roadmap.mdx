## 마이그레이션 전략: 전면 교체 vs 병행

Claude Code에서 Codex로 전환할 때 두 가지 접근이 가능합니다.

| 전략 | 설명 | 권장 대상 |
|------|------|-----------|
| **전면 교체** | Codex로 완전히 이동, Claude Code 사용 중단 | 개인 개발자, 작은 팀 |
| **병행 사용** | 프로젝트 성격에 따라 선택적으로 사용 | 팀 환경, 기존 자동화 많은 경우 |

이 로드맵은 **전면 교체** 를 기준으로 작성되었으나, 병행 사용 시에도 동일한 단계로 적용할 수 있습니다.

## Day 1: 환경 구축과 첫 실행

첫날의 목표는 "Codex가 내 기계에서 동작한다"를 확인하는 것입니다.

### 체크리스트

- [ ] **Codex CLI 설치**
  ```bash
  npm install -g @openai/codex
  codex --version
  ```
- [ ] **인증 설정** (API 키 또는 ChatGPT 계정)
  ```bash
  export OPENAI_API_KEY="sk-..."
  # 또는 codex login (ChatGPT OAuth)
  ```
- [ ] **기본 `config.toml` 생성**
  ```toml
  # ~/.codex/config.toml
  model = "gpt-5.4"
  approval_policy = "on-request"
  sandbox_mode = "workspace-write"
  ```
- [ ] **첫 번째 프롬프트 실행** — 기존 프로젝트에서 간단한 작업 시도
  ```bash
  cd ~/my-project
  codex "README.md 를 읽고 프로젝트 구조를 설명해 줘"
  ```
- [ ] **`AGENTS.md` 기초 작성** — 기존 `CLAUDE.md` 내용을 복사하여 시작
  ```bash
  cp CLAUDE.md AGENTS.md
  ```

:::tip[팁]
`CLAUDE.md` 내용을 그대로 `AGENTS.md` 로 복사해도 대부분 잘 동작합니다. 두 파일 모두 자연어 지침이기 때문입니다. 세부 조정은 Week 1에서 합니다.
:::

## Week 1: 핵심 워크플로 전환

일주일의 목표는 "일상 개발 작업을 Codex로 할 수 있다"입니다.

### 체크리스트

- [ ] **모델 확인 및 조정**
  ```toml
  # API 키 사용자
  model = "gpt-5.4"
  model_reasoning_effort = "medium"

  # ChatGPT Pro 사용자
  model = "gpt-5.5"
  ```
- [ ] **승인/sandbox 정책 튜닝** — 너무 많은 승인 요청이 불편하다면 조정
  ```toml
  approval_policy = "never"   # 자동화 환경
  sandbox_mode = "workspace-write"
  ```
- [ ] **MCP 서버 마이그레이션** — `.mcp.json` / `mcpServers` → `[mcp_servers.<id>]`
  ```toml
  [mcp_servers.github]
  command = "npx"
  args = ["-y", "@modelcontextprotocol/server-github"]
  enabled = true
  ```
- [ ] **AGENTS.md 정제** — Claude Code 전용 문법 제거, Codex에 맞게 조정
- [ ] **slash 커맨드 익히기** — `/model`, `/permissions`, `/review`, `/agent`
- [ ] **`codex exec` 비대화형 실행 테스트**
  ```bash
  codex exec "테스트 실패 원인을 분석하고 수정해 줘"
  ```

:::note[참고]
Claude Code의 `claude -p "..."` 는 Codex의 `codex exec "..."` (또는 `codex e "..."`)에 대응합니다. CI/CD 파이프라인 업데이트 시 이 점을 반영하세요.
:::

- [ ] **세션 재개 확인**
  ```bash
  codex resume --last "이어서 계속해 줘"
  ```

## Week 2: 고급 기능 적용

2주차 목표는 "Claude Code에서 사용하던 고급 기능을 Codex로 재현한다"입니다.

### 체크리스트

- [ ] **Hooks 마이그레이션** (`features.codex_hooks = true` 게이트 필요)
  ```toml
  # ~/.codex/config.toml
  [features]
  codex_hooks = true

  [[hooks.PreToolUse]]
  matcher = "^Bash$"
  [[hooks.PreToolUse.hooks]]
  type = "command"
  command = "/usr/bin/python3 .codex/hooks/policy.py"
  timeout = 30
  ```
- [ ] **Subagents 설정**
  ```toml
  # ~/.codex/agents/reviewer.toml
  name = "reviewer"
  description = "코드 리뷰 전문 에이전트"
  model = "gpt-5.4"
  model_reasoning_effort = "high"
  sandbox_mode = "read-only"
  ```
- [ ] **Skills 설치** — Claude Code skills을 Agent Skills 표준으로 전환
  ```bash
  # 공식 skill 설치
  codex skill-installer linear
  codex skill-installer vercel-deploy
  ```
- [ ] **Desktop App 설치 및 연동** (선택)
  - macOS / Windows 앱 설치
  - CLI 세션이 그대로 연동되는지 확인
  - Auto Tasks 설정
- [ ] **Codex Rules 검토** — `developers.openai.com/codex/rules`
  프로젝트·팀 단위 규칙 정의로 AGENTS.md와 보완
- [ ] **CI/CD 파이프라인 업데이트**
  ```yaml
  # GitHub Actions 예시
  - name: Codex 자동 수정
    run: codex exec --ask-for-approval never "린트 오류 수정"
    env:
      OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
  ```

## 마이그레이션 후 검증

전환이 완료되면 다음 항목을 확인하세요.

| 항목 | 확인 방법 |
|------|-----------|
| 기본 대화 | `codex "Hello"` 응답 확인 |
| 파일 읽기/쓰기 | 간단한 파일 수정 작업 |
| MCP 도구 | `/permissions` 에서 MCP 서버 목록 확인 |
| 세션 재개 | `codex resume --last` |
| 비대화형 | `codex exec "ls -la"` |
| Skills | `$skill-creator` 로 skill 목록 확인 |

:::caution[주의]
기존 Claude Code hooks (PreToolUse 등)는 Codex에서 직접 동작하지 않습니다. `features.codex_hooks = true` 를 활성화하고 Codex hooks 형식으로 재작성해야 합니다. 자세한 내용은 [05-feature-mapping](/learn-codex-from-claude-code/05-feature-mapping/) 섹션을 참고하세요.
:::

## 핵심 정리

- **Day 1**: CLI 설치, 인증, 기본 `config.toml`, `AGENTS.md` 복사로 첫 실행 확인
- **Week 1**: 모델 튜닝, MCP 마이그레이션, `AGENTS.md` 정제, `codex exec` 검증
- **Week 2**: Hooks, Subagents, Skills, Desktop App, CI/CD 파이프라인 업데이트
- `CLAUDE.md` 는 `AGENTS.md` 로 복사하는 것부터 시작하면 빠릅니다
- `claude -p "..."` 는 `codex exec "..."` 로 대체합니다
- Hooks 마이그레이션은 `features.codex_hooks = true` 게이트가 필요합니다
