## 마이그레이션 체크리스트

아래 항목을 하나씩 확인하며 마이그레이션 완료 여부를 점검하세요.

### 1. 설치 및 인증

- [ ] Codex CLI가 설치되어 있다 (`codex --version` 실행 확인)
- [ ] OpenAI 계정으로 인증 완료 (`codex auth status` 확인)
- [ ] API Key 또는 ChatGPT 인증 중 하나 이상 설정되어 있다
- [ ] 원하는 모델이 현재 플랜에서 사용 가능한지 확인했다

### 2. 글로벌 설정

- [ ] `~/.codex/config.toml`에 기본 모델과 reasoning effort가 설정되어 있다
- [ ] `approval_policy`와 `sandbox_mode`가 작업 성격에 맞게 설정되어 있다
- [ ] `~/.codex/AGENTS.md`에 개인 선호 지침이 작성되어 있다 (선택)
- [ ] 필요하다면 Profiles (`[profiles.<name>]`)가 정의되어 있다

### 3. 프로젝트 설정

- [ ] 프로젝트 루트에 `AGENTS.md`가 작성되어 있다
- [ ] `.codex/config.toml`에 프로젝트별 설정이 정의되어 있다
- [ ] project trust를 승인했다 (처음 실행 시 `y` 입력)
- [ ] `AGENTS.md` 내용이 에이전트에 반영되는 것을 확인했다

### 4. Permissions / Approval 정책

- [ ] `approval_policy`가 프로젝트 요구사항에 맞게 설정되어 있다
- [ ] `sandbox_mode`가 필요한 파일 접근 수준을 허용한다
- [ ] 팀 환경이라면 Codex Rules를 통해 공통 정책이 정의되어 있다
- [ ] `--yolo`를 습관적으로 사용하지 않도록 주의하고 있다

### 5. Skills 이전

- [ ] Claude Code `.claude/skills/`의 스킬 목록을 파악했다
- [ ] 이전할 스킬의 `SKILL.md`를 Agent Skills 표준으로 변환했다
  - Claude Code 전용 frontmatter 필드 제거 (`disable-model-invocation` 등)
  - `name`, `description` 필수 필드 유지
- [ ] 스킬을 `~/.agents/skills/<name>/` 또는 `.agents/skills/<name>/`에 배치했다
- [ ] `codex`에서 `/skills`로 스킬이 인식되는 것을 확인했다

### 6. Hooks 이전

- [ ] Claude Code `settings.json`의 hooks 목록을 파악했다
- [ ] `config.toml`에 `[features] codex_hooks = true`를 설정했다
- [ ] hooks를 `[[hooks.PreToolUse]]` 등 Codex 형식으로 변환했다
- [ ] hooks가 실제로 실행되는 것을 `~/.codex/log/codex-tui.log`로 확인했다

### 7. MCP 이전

- [ ] Claude Code `.mcp.json`의 MCP 서버 목록을 파악했다
- [ ] `config.toml`의 `[mcp_servers.<id>]` 형식으로 변환했다
- [ ] 각 서버의 인증 토큰/환경 변수가 올바르게 설정되어 있다
- [ ] `codex`에서 MCP 도구가 정상 동작하는 것을 확인했다

### 8. Subagents / CI

- [ ] 필요한 에이전트 정의를 `.codex/agents/<name>.toml`로 작성했다
- [ ] `[agents]` 한도 (`max_threads`, `max_depth`, `job_max_runtime_seconds`)가 설정되어 있다
- [ ] CI 환경(GitHub Actions 등)에서 `codex exec --json` 파이프라인을 테스트했다
- [ ] `history.persistence` 설정이 세션 재개 요구사항에 맞게 설정되어 있다

### 9. 팀 환경 (해당하는 경우)

- [ ] `.codex/config.toml`을 저장소에 체크인했다 (민감 정보 미포함 확인)
- [ ] `AGENTS.md`가 저장소에 체크인되어 팀원 모두가 사용할 수 있다
- [ ] 팀 공용 스킬이 `.agents/skills/`에 체크인되어 있다
- [ ] `AGENTS.override.md`가 `.gitignore`에 추가되어 있다

### 10. 프롬프트와 검증 루프

- [ ] 자주 쓰는 프롬프트가 Goal / Context / Constraints / Done when 구조를 따른다
- [ ] 복잡하거나 모호한 작업은 `/plan` 또는 `Shift`+`Tab`으로 먼저 계획하게 한다
- [ ] 버그 수정 요청에는 재현 절차와 기대 결과를 포함한다
- [ ] 기능 구현 요청에는 최소 검증 명령 또는 수동 확인 절차를 포함한다
- [ ] 반복되는 팀 규칙은 `AGENTS.md`로 옮겼고, 반복되는 작업 절차는 Skill 후보로 정리했다
