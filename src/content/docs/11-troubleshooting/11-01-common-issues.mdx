## Q1. Sandbox가 명령 실행을 거부합니다

**증상:** `codex`가 파일 수정이나 특정 셸 명령을 실행하려 하면 "Permission denied" 또는 sandbox 거부 메시지가 나타납니다.

**원인과 해결:**

현재 `sandbox_mode`가 작업에 비해 제한적으로 설정되어 있을 가능성이 높습니다.

```toml
# ~/.codex/config.toml 또는 .codex/config.toml
# 파일 읽기만 허용 (코드 수정 불가)
sandbox_mode = "read-only"

# ↓ 아래로 변경

# 워크스페이스 쓰기 허용 (일반 개발 작업)
sandbox_mode = "workspace-write"

# 모든 접근 허용 (주의해서 사용)
sandbox_mode = "danger-full-access"
```

일시적으로 더 넓은 권한이 필요하면 실행 시 플래그로 지정할 수 있습니다.

```bash
codex -s workspace-write "파일 수정 작업..."
```

## Q2. MCP 서버 인증에 실패합니다

**증상:** `[mcp_servers.xxx]` 를 설정했는데 "Authentication failed" 또는 "Connection refused" 오류가 납니다.

**원인과 해결:**

1. **환경 변수 미설정** — `env` 키의 값이 실제로 셸에 있는지 확인합니다.

```bash
# 환경 변수 확인
echo $GITHUB_TOKEN
echo $LINEAR_API_KEY
```

2. **토큰 형식 오류** — `bearer_token_env_var`를 사용하는 서버는 Bearer 토큰 형식이 필요합니다.

```toml
[mcp_servers.my-api]
command = "npx"
args = ["-y", "@some/mcp-server"]
bearer_token_env_var = "MY_API_TOKEN"
# MY_API_TOKEN 환경 변수가 실제로 설정되어 있어야 함
```

3. **서버 미설치** — `npx` 기반 MCP 서버는 처음 실행 시 패키지 설치가 필요합니다. 수동으로 한 번 실행해 설치를 완료하세요.

```bash
npx -y @modelcontextprotocol/server-github
```

## Q3. AGENTS.md가 반영되지 않습니다

**증상:** `AGENTS.md`를 작성했는데 에이전트가 내용을 무시하는 것 같습니다.

**원인과 해결:**

1. **project trust 미승인** — 프로젝트 `.codex/config.toml` 또는 `AGENTS.md`가 처음 로드될 때 trust 프롬프트가 나타납니다. `y`를 입력하지 않으면 프로젝트 파일이 로드되지 않습니다.

```
This project contains a .codex/config.toml file.
Do you want to trust and load it? [y/N]  ← 반드시 y 입력
```

2. **파일 크기 초과** — 기본 `project_doc_max_bytes = 32768` (32 KiB)를 초과하면 잘립니다.

```toml
# .codex/config.toml
project_doc_max_bytes = 65536  # 64 KiB로 증가
```

3. **우선순위 혼동** — 파일 로드 우선순위를 확인합니다.

```
~/.codex/AGENTS.override.md  (최우선)
~/.codex/AGENTS.md
프로젝트 루트 AGENTS.override.md
프로젝트 루트 AGENTS.md        (기본)
```

## Q4. 실행할 때마다 project trust 프롬프트가 나타납니다

**증상:** 같은 프로젝트에서 `codex`를 실행할 때마다 trust 여부를 묻습니다.

**원인과 해결:**

trust 결정은 세션 단위로 기억됩니다. 영구적으로 특정 프로젝트를 신뢰하려면 글로벌 설정에 등록합니다.

```toml
# ~/.codex/config.toml
[[trusted_projects]]
path = "/Users/me/my-project"
```

또는 `codex` 실행 시 `--trust-project` 플래그를 사용합니다.

```bash
codex --trust-project
```

## Q5. .agents/skills/ 의 스킬이 인식되지 않습니다

**증상:** `~/.agents/skills/my-skill/SKILL.md`를 만들었는데 `/skills` 목록에 나타나지 않습니다.

**원인과 해결:**

1. **경로 오류** — `~/.codex/skills/`가 아닌 `~/.agents/skills/`가 올바른 경로입니다.

```bash
# 잘못된 위치
ls ~/.codex/skills/

# 올바른 위치
ls ~/.agents/skills/
```

2. **디렉토리명과 name 필드 불일치** — 디렉토리 이름이 `SKILL.md`의 `name` 필드와 정확히 일치해야 합니다.

```
~/.agents/skills/
└── code-review/          ← 디렉토리명
    └── SKILL.md
```

```markdown
---
name: code-review         ← name 필드가 디렉토리명과 동일해야 함
description: "..."
---
```

3. **비활성화 설정 확인** — `config.toml`에서 해당 스킬 경로가 `enabled = false`로 되어 있지 않은지 확인합니다.

## Q6. gpt-5.5 모델을 API 키로 사용하려 하면 오류가 납니다

**증상:** `OPENAI_API_KEY`를 설정하고 `model = "gpt-5.5"`를 지정했는데 "Model not available" 오류가 납니다.

**원인과 해결:**

`gpt-5.5`는 **ChatGPT 계정 인증** 전용 모델입니다. API Key 방식으로는 사용할 수 없습니다.

| 모델 | API Key | ChatGPT 인증 |
|------|---------|-------------|
| `gpt-5.5` | 불가 | 가능 |
| `gpt-5.4` | 가능 | 가능 |
| `gpt-5.4-mini` | 가능 | 가능 |
| `gpt-5.3-codex-spark` | 불가 | ChatGPT Pro only |

API Key 환경에서는 `gpt-5.4`를 사용하세요.

```toml
# API Key 환경
model = "gpt-5.4"

# ChatGPT 인증 환경
model = "gpt-5.5"
```

## Q7. Hooks가 실행되지 않습니다

**증상:** `config.toml`에 hooks를 정의했는데 아무 반응이 없습니다.

**원인과 해결:**

`features.codex_hooks = true` 게이트가 활성화되어 있는지 가장 먼저 확인합니다.

```toml
# config.toml
[features]
codex_hooks = true          # ← 이 줄이 없으면 hooks 전체가 무시됨

[[hooks.PreToolUse]]
matcher = "^Bash$"
[[hooks.PreToolUse.hooks]]
type = "command"
command = "/usr/bin/python3 .codex/hooks/policy.py"
timeout = 30
```

게이트를 활성화한 후에도 동작하지 않으면 `matcher` 정규식이 올바른지 확인하세요. `^Bash$` 는 정확히 "Bash" 도구에만 매칭됩니다.

<ConceptCard title="로그로 hooks 동작 확인">
hooks 실행 여부를 확인하려면 `~/.codex/log/codex-tui.log` 파일을 확인합니다.

```bash
tail -f ~/.codex/log/codex-tui.log
```

hooks가 실행되면 로그에 관련 항목이 기록됩니다.
</ConceptCard>

## 핵심 정리

- sandbox 거부는 `sandbox_mode` 설정을 확인하고 `-s workspace-write` 로 일시 변경할 수 있습니다
- MCP 인증 실패 시 환경 변수 설정과 `bearer_token_env_var` 값을 점검하세요
- AGENTS.md가 반영되지 않으면 **project trust 승인** 여부를 먼저 확인하세요
- `gpt-5.5`는 API Key 환경에서 사용할 수 없습니다. API Key 환경에서는 `gpt-5.4`를 사용하세요
- hooks 미작동의 가장 흔한 원인은 `features.codex_hooks = true` 누락입니다
- User-level skills 경로는 `~/.agents/skills/` 이며, `~/.codex/skills/`가 아닙니다
