## 세 가지 인터페이스

Codex를 사용하는 주요 인터페이스는 세 가지입니다. 각각 다른 사용 맥락에 맞게 설계되었습니다.

| 인터페이스 | 주 사용 환경 | 특징 |
|------------|-------------|------|
| **Claude Code TUI** | 터미널 | 키보드 중심, 단일 세션 |
| **Codex TUI** | 터미널 | 키보드 중심, 멀티 에이전트 스레드 |
| **Codex Desktop App** | macOS / Windows | GUI, 멀티 에이전트 시각화 |

## Claude Code TUI vs Codex TUI

터미널 인터페이스 관점에서 두 도구는 전체적으로 유사합니다. 둘 다 프롬프트를 입력하고 에이전트의 응답과 도구 실행을 터미널에서 확인합니다.

### 키보드 단축키 비교

| 동작 | Claude Code | Codex |
|------|-------------|-------|
| 대화 초기화 | `/clear` | `/clear` 또는 Ctrl+L |
| 이전 입력 탐색 | 위/아래 방향키 | 위/아래 방향키 (드래프트) |
| 히스토리 검색 | — | Ctrl+R |
| 클립보드 복사 | — | Ctrl+O |
| 자동완성 | Tab | Tab (예약됨, 다름) |
| 에이전트 전환 | — | `/agent` |

:::caution[주의]
Codex TUI에서 Tab 키는 Claude Code와 다르게 동작합니다. Claude Code에서 Tab으로 파일 경로를 자동완성하던 습관이 있다면 주의하세요.
:::

### /personality 커맨드

Codex TUI에는 Claude Code에 없는 `/personality` 커맨드가 있습니다. 에이전트의 응답 스타일을 전환합니다.

| 모드 | 설명 |
|------|------|
| `concise` | 간결하고 핵심만 전달 |
| `conversational` | 상세하고 대화적 |

```
/personality concise
→ 에이전트가 짧고 간결하게 답변
```

## Codex Desktop App

Desktop App은 TUI와 근본적으로 다른 경험을 제공합니다. 2026년 3월 macOS 출시 후 같은 달 Windows로 확장되었습니다.

### TUI와 Desktop App의 차이

| 기능 | TUI | Desktop App |
|------|-----|-------------|
| 에이전트 수 | 순차 (스레드 전환) | **동시 다중 에이전트 시각화** |
| Git 작업 | 현재 브랜치에서 | **워크트리 격리** (로컬 git 영향 없음) |
| diff 검토 | 텍스트 출력 | **GUI diff 뷰어** + 코멘트 |
| Skills 관리 | CLI 명령 | **전용 Skills UI** |
| 자동 작업 | 수동 실행 | **Auto Tasks** (스케줄 기반) |
| 세션 전환 | `codex resume` | 프로젝트별 스레드 GUI |

<ConceptCard title="워크트리 격리란?">
Desktop App은 각 에이전트에게 현재 레포지토리의 **격리된 git 워크트리 복사본** 을 제공합니다.

이 덕분에:
- 에이전트 A가 `feature-a` 브랜치에서 작업하는 동안 에이전트 B는 `feature-b` 에서 동시 작업 가능
- 에이전트 실수가 로컬 working directory에 영향을 주지 않음
- 검토 후 마음에 들면 merge, 아니면 폐기

Claude Code에서 oh-my-claudecode 같은 오케스트레이터로 구현하던 워크트리 패턴을 Desktop App이 기본 지원합니다.
</ConceptCard>

### Auto Tasks

Desktop App의 Auto Tasks는 스케줄 기반 백그라운드 자동 작업입니다.

| 자동 작업 예시 |
|----------------|
| 매일 아침 CI 실패 요약 생성 |
| 새 이슈 분류 및 라벨링 |
| 일일 릴리스 브리프 작성 |
| 의존성 업데이트 PR 자동 생성 |

이는 Claude Code에 직접 대응하는 기능이 없는 Codex Desktop App 전용 기능입니다.

### CLI와 Desktop App의 연동

Desktop App은 별도 설정 없이 CLI의 세션과 설정을 그대로 활용합니다.

```
CLI에서 설정한 ~/.codex/config.toml
     ↓
Desktop App에서 자동으로 동일한 모델·정책 적용

CLI에서 진행 중인 세션
     ↓
Desktop App에서 이어서 확인·수정 가능
```

## 세 인터페이스 선택 가이드

| 상황 | 권장 인터페이스 |
|------|----------------|
| SSH 원격 서버 작업 | Codex TUI |
| 빠른 단일 작업 | Codex TUI |
| 여러 기능을 동시에 개발 | Codex Desktop App |
| 에이전트 결과를 신중히 검토하고 싶다 | Codex Desktop App |
| 정기 반복 작업 자동화 | Codex Desktop App (Auto Tasks) |
| CI/CD 파이프라인 | `codex exec` (비대화형) |

## 핵심 정리

- **Codex TUI** 는 Claude Code TUI와 유사하지만 `/agent` 스레드 전환과 `/personality` 가 추가됩니다
- Tab 키 동작이 다르므로 Claude Code에서 Tab 자동완성에 익숙하다면 주의가 필요합니다
- **Desktop App** 은 워크트리 격리, 동시 다중 에이전트 시각화, GUI diff 검토를 제공합니다
- **Auto Tasks** 는 Desktop App 전용 스케줄 기반 백그라운드 작업 기능입니다
- Desktop App은 CLI 설정(`~/.codex/config.toml`)과 세션을 **그대로 공유** 합니다
- 단순 터미널 작업은 TUI, 복잡한 멀티 에이전트 작업은 Desktop App이 적합합니다
