## 전체 대응 표

아래 표는 Claude Code의 주요 개념을 Codex와 1:1로 대응시킨 것입니다. 완전히 동일하지 않은 항목은 비고에 표시했습니다.

| 카테고리 | Claude Code | Codex | 비고 |
|----------|-------------|-------|------|
| **CLI 시작** | `claude` | `codex` | |
| **비대화형 실행** | `claude -p "..."` | `codex exec "..."` 또는 `codex e "..."` | |
| **대화 재개** | `claude -c` | `codex resume --last` | |
| **특정 세션 재개** | `claude -r <id>` | `codex resume <SESSION_ID>` | |
| **이미지 입력** | — | `codex -i image.png "..."` | Codex에만 있음 |
| **홈 디렉토리** | `~/.claude/` | `~/.codex/` | |
| **글로벌 설정** | `~/.claude/settings.json` | `~/.codex/config.toml` | 형식 다름 (JSON vs TOML) |
| **프로젝트 설정** | `.claude/settings.json` | `.codex/config.toml` | |
| **로컬 설정** | `.claude/settings.local.json` | — | Codex에 없음 |
| **지침 파일 (글로벌)** | `~/.claude/CLAUDE.md` | `~/.codex/AGENTS.md` | |
| **지침 파일 (프로젝트)** | `./CLAUDE.md` | `./AGENTS.md` | |
| **지침 오버라이드** | `./CLAUDE.local.md` | `./AGENTS.override.md` | |
| **Subagent 정의** | `.claude/agents/<name>.md` | `~/.codex/agents/<name>.toml` 또는 `.codex/agents/<name>.toml` | 형식 다름 |
| **Skills 디렉토리** | `.claude/skills/<name>/SKILL.md` | `$HOME/.agents/skills/<name>/SKILL.md` | 경로 다름 |
| **MCP 설정** | `~/.claude.json` 또는 `.mcp.json` 의 `mcpServers` 키 | `~/.codex/config.toml` 의 `[mcp_servers.<id>]` | |
| **Hooks 설정** | `~/.claude/settings.json` 의 `hooks` 키 | `~/.codex/hooks.json` 또는 `config.toml` 의 `[hooks]` | |
| **세션 저장** | 내부 관리 | `~/.codex/sessions/` | |
| **히스토리** | — | `~/.codex/history.jsonl` | |
| **로그** | — | `~/.codex/log/codex-tui.log` | |

## 카테고리별 상세 대응

### CLI 명령어

| Claude Code | Codex | 설명 |
|-------------|-------|------|
| `claude` | `codex` | 대화형 TUI 시작 |
| `claude -p "..."` | `codex exec "..."` | 비대화형 단일 작업 |
| `claude -c` | `codex resume --last` | 마지막 세션 재개 |
| `claude -r <id>` | `codex resume <id>` | 특정 세션 재개 |
| — | `codex fork` | 현재 세션 분기 (Codex 전용) |
| — | `codex exec --json` | JSONL 출력 (Codex 전용) |

### 모델 설정

| Claude Code | Codex |
|-------------|-------|
| `"model": "claude-sonnet-4-6"` (JSON) | `model = "gpt-5.4"` (TOML) |
| `/model` slash 커맨드 | `/model` slash 커맨드 |
| `/effort` slash 커맨드 | `model_reasoning_effort = "low\|medium\|high"` |

### 권한·승인

| Claude Code | Codex |
|-------------|-------|
| `"defaultMode": "default"` | `approval_policy = "on-request"` |
| `"defaultMode": "auto"` | `approval_policy = "never"` |
| `"defaultMode": "plan"` | — (Codex에 Plan Mode 없음) |
| `"allow": ["Bash(npm *)"]` | `approval_policy = "granular"` + 규칙 |
| `"deny": ["Bash(curl *)"]` | sandbox로 네트워크 제한 |
| `bypassPermissions` | `--dangerously-bypass-approvals-and-sandbox` (`--yolo`) |

<ConceptCard title="Claude Code에만 있는 기능들">
다음은 Claude Code에 있지만 Codex에 직접적인 대응이 없는 기능입니다:

- **Plan Mode** (`/plan`, `ExitPlanMode`): 실행 없이 계획만 수립하는 전용 모드
- `/memory`: 대화 중 즉시 CLAUDE.md 편집
- `/compact`: 컨텍스트 압축
- `/loop`: 반복 실행 루프
- `.claude/settings.local.json`: 로컬 오버라이드 설정 파일

이 기능들은 Codex에서 다른 방식으로 구현하거나 우회해야 합니다.
</ConceptCard>

### Skills 경로

| Claude Code | Codex |
|-------------|-------|
| `.claude/skills/<name>/SKILL.md` | `$HOME/.agents/skills/<name>/SKILL.md` (USER) |
| — | `/etc/codex/skills/` (ADMIN) |
| — | `$REPO_ROOT/.agents/skills/` (REPO root) |
| — | `$CWD/.agents/skills/` (REPO CWD) |

:::caution[주의]
Codex skills 경로는 `.codex/skills/` 가 **아닙니다**. `$HOME/.agents/skills/` 또는 `.agents/skills/` (프로젝트 내)에 위치해야 합니다.
:::

## 핵심 정리

- `claude -p "..."` 는 `codex exec "..."` 로 대체됩니다
- 설정 파일은 JSON(`settings.json`) 에서 TOML(`config.toml`) 로 바뀝니다
- 지침 파일은 `CLAUDE.md` 에서 `AGENTS.md` 로 바뀝니다
- Skills 경로는 `.claude/skills/` 가 아닌 **`.agents/skills/`** 입니다
- Claude Code의 **Plan Mode** 와 여러 slash 커맨드는 Codex에 대응 기능이 없습니다
- Hooks는 v0.124.0+ 에서 stable이며 `PreToolUse`, `PostToolUse`, `Stop` 을 지원합니다 (게이트 불필요)
