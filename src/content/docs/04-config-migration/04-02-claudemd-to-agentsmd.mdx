Claude Code에서 `CLAUDE.md`는 AI에게 프로젝트 컨텍스트와 행동 지침을 제공하는 핵심 파일이었습니다. Codex에서는 같은 역할을 `AGENTS.md`가 담당합니다. 파일명만 바뀐 것이 아니라 **우선순위 체계** 와 **로딩 방식** 도 달라졌습니다.

## 파일 위치와 우선순위 비교

<Compare>
  <Fragment slot="left">
  **Claude Code 우선순위 (낮음 → 높음)**
  1. `~/.claude/CLAUDE.md` (user)
  2. `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` (project)
  3. `./CLAUDE.local.md` (local, git 제외)

  </Fragment>
  <Fragment slot="right">
  **Codex 로딩 순서**
  1. 글로벌: `~/.codex/AGENTS.override.md`가 있으면 그것만, 없으면 `~/.codex/AGENTS.md`
  2. 프로젝트: 프로젝트 루트 → CWD까지 각 디렉터리에서 `AGENTS.override.md`, `AGENTS.md`, fallback 이름 순으로 하나씩 선택
  3. 병합: 선택된 파일들을 루트에서 CWD 방향으로 concat

  </Fragment>
</Compare>

:::caution[중요한 차이]
Codex는 루트에서 CWD까지 instruction 파일을 **순서대로 합쳐서** 읽습니다. 하위 디렉터리의 파일이 뒤에 붙기 때문에 상위 파일보다 우선됩니다. `AGENTS.override.md`는 같은 디렉터리에서 `AGENTS.md`보다 먼저 선택되는 대체 파일이며, 글로벌 override가 프로젝트 지침보다 항상 높다는 뜻은 아닙니다.
:::

## 전체 우선순위 순서

```
먼저 글로벌에서 하나 선택:
  ~/.codex/AGENTS.override.md
  또는 ~/.codex/AGENTS.md

그 다음 프로젝트 루트부터 CWD까지 각 디렉터리에서 하나씩 선택:
  AGENTS.override.md → AGENTS.md → fallback filenames

선택된 파일들을 이 순서대로 concat:
  글로벌 → 프로젝트 루트 → ... → CWD
```

## 주요 설정 옵션

### project_doc_max_bytes

AGENTS.md 파일의 최대 로드 크기를 제한합니다. 기본값은 32KiB입니다.

```toml
# ~/.codex/config.toml
project_doc_max_bytes = 32768  # 32 KiB (기본값)
```

대용량 AGENTS.md를 사용하는 경우 값을 늘릴 수 있지만, 컨텍스트 예산을 고려해야 합니다.

### project_doc_fallback_filenames

`AGENTS.md`라는 파일명 대신 다른 파일명을 사용하고 싶을 때 설정합니다.

```toml
project_doc_fallback_filenames = ["TEAM_GUIDE.md", ".agents.md"]
```

이 설정이 있으면 Codex는 `AGENTS.md`를 찾지 못했을 때 지정된 파일명으로 대체 탐색합니다.

## 실전 변환 예시

### 기존 CLAUDE.md

```markdown
# 프로젝트 가이드

## 코딩 규칙
- TypeScript strict 모드 사용
- 함수형 컴포넌트만 사용 (클래스 컴포넌트 금지)
- 테스트 파일은 `__tests__` 디렉터리에 배치

## 커밋 규칙
- Conventional Commits 형식 준수
- 커밋 전 `pnpm lint && pnpm test` 실행

## 금지 사항
- `console.log` 프로덕션 코드에 사용 금지
- 직접 DOM 조작 금지
```

### 변환된 AGENTS.md

```markdown
# 프로젝트 가이드

## 코딩 규칙
- TypeScript strict 모드 사용
- 함수형 컴포넌트만 사용 (클래스 컴포넌트 금지)
- 테스트 파일은 `__tests__` 디렉터리에 배치

## 커밋 규칙
- Conventional Commits 형식 준수
- 커밋 전 `pnpm lint && pnpm test` 실행

## 금지 사항
- `console.log` 프로덕션 코드에 사용 금지
- 직접 DOM 조작 금지
```

:::tip[내용은 거의 그대로 이전 가능]
AGENTS.md의 본문 내용은 자연어 마크다운이므로 CLAUDE.md 내용을 거의 그대로 복사해 사용할 수 있습니다. 파일명만 바꾸면 기본 동작은 유지됩니다.
:::

## 디렉터리별 AGENTS.md 활용

Codex의 concat 방식을 활용하면 디렉터리별로 세분화된 지침을 줄 수 있습니다.

```
project/
├── AGENTS.md          ← 프로젝트 전체 공통 규칙
├── frontend/
│   └── AGENTS.md      ← 프론트엔드 특화 규칙 (추가됨)
└── backend/
    └── AGENTS.md      ← 백엔드 특화 규칙 (추가됨)
```

CWD가 `frontend/`일 때는 프로젝트 루트 + `frontend/` AGENTS.md가 합쳐져 로드됩니다.

<ConceptCard title="AGENTS.override.md 활용 시나리오">
팀원들이 공유하는 `AGENTS.md`는 git에 커밋하고, 개인 규칙은 `~/.codex/AGENTS.override.md`에 작성하세요. 개인 override는 모든 프로젝트에서 최우선 적용되므로 팀 규칙을 유지하면서 개인 작업 스타일을 추가할 수 있습니다.
</ConceptCard>

## AGENTS.md — 공유 가능한 지침 파일

`AGENTS.md` 는 Codex 전용 파일이 아닙니다. 공식 AGENTS.md 사이트와 OpenAI Codex 문서는 여러 에이전트가 읽을 수 있는 프로젝트 지침 파일로 이 이름을 안내합니다.

:::tip[한 번 작성, 모든 도구에서 동작]
`AGENTS.md` 를 프로젝트에 추가해두면 Codex는 작업 시작 시 이를 읽습니다. 다른 AI 코딩 도구와 함께 쓰는 팀이라면 각 도구의 현재 지원 여부를 확인한 뒤, 공통 지침은 `AGENTS.md`에 두고 도구별 예외만 별도 파일로 분리하세요.
:::

## OS 수준 샌드박스

Codex의 샌드박스는 파일 시스템과 네트워크 접근 같은 실행 권한을 제한합니다. AGENTS.md는 모델 지침이고, 실제 파일·명령 실행 차단은 `sandbox_mode`, permissions, approval policy가 담당합니다.

| OS | 샌드박스 기술 |
|----|--------------|
| macOS | Apple Seatbelt 프레임워크 (커널 레벨) |
| Linux | Landlock + seccomp |

:::note[참고]
이 샌드박스는 `sandbox_mode = "workspace-write"` 설정과 연동되어, 에이전트가 작업 디렉터리 외부를 수정하거나 허가되지 않은 네트워크 접근을 시도할 때 커널이 직접 차단합니다.
:::

## 핵심 정리

- `CLAUDE.md` → `AGENTS.md`로 파일명이 변경되었으며 내용은 대부분 그대로 이전 가능합니다
- `AGENTS.md` 는 Codex가 작업 전에 읽는 공유 가능한 프로젝트 지침 파일입니다
- Codex는 루트에서 CWD까지 모든 `AGENTS.md`를 concat하므로 디렉터리별 세분화가 가능합니다
- `AGENTS.override.md`는 개인 또는 팀의 최우선 오버라이드 파일입니다
- `project_doc_max_bytes`로 로드 크기를 제한하고, `project_doc_fallback_filenames`로 대체 파일명을 지정할 수 있습니다
- 글로벌 `~/.codex/AGENTS.override.md`는 모든 프로젝트에서 가장 높은 우선순위를 가집니다
- 샌드박스와 approval policy는 AGENTS.md 지침과 별개로 파일·명령 실행 권한을 제한합니다
