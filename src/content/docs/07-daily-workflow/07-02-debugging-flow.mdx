## 디버깅에서 멀티 에이전트가 필요한 이유

복잡한 버그는 대개 여러 파일과 모듈에 걸쳐 있습니다. 단일 대화 컨텍스트에서 모든 가설을 순차적으로 검증하면 시간이 오래 걸리고 컨텍스트 창도 빠르게 소모됩니다.

Claude Code에서는 `Task` 도구와 `.claude/agents/` 서브에이전트로 이 문제를 해결했습니다. Codex는 `spawn_agent`, `/agent` 슬래시, `codex fork` 로 유사한 흐름을 지원합니다.

## spawn_agent로 병렬 디버깅

Codex에서 모델이 `spawn_agent` 도구를 호출하면 새 에이전트 스레드가 생성됩니다. 메인 스레드는 계속 실행되며, 서브에이전트가 완료되면 결과를 돌려받습니다.

```toml
# ~/.codex/agents/debugger.toml
name = "debugger"
description = "로그와 스택 트레이스를 분석하여 근본 원인을 파악합니다"
model = "gpt-5.4"
model_reasoning_effort = "high"
sandbox_mode = "read-only"
developer_instructions = """
주어진 오류와 스택 트레이스를 분석하세요.
가설을 3가지 제시하고 각 가설의 검증 방법을 설명하세요.
"""
```

에이전트 한도는 config.toml에서 설정합니다.

```toml
[agents]
max_threads = 6
max_depth = 1
job_max_runtime_seconds = 1800
```

:::note[참고]
`max_depth = 1` 은 에이전트가 또 다른 에이전트를 생성하는 것을 막습니다. 복잡한 디버깅 트리가 필요한 경우 값을 늘릴 수 있지만, 실행 비용과 복잡도가 함께 증가합니다.
:::

## /agent 슬래시로 스레드 전환

TUI에서 `/agent` 를 입력하면 현재 실행 중인 에이전트 스레드 목록이 표시됩니다. 다른 스레드를 선택하면 해당 스레드의 컨텍스트로 즉시 전환됩니다.

```
/agent
  > [0] main — 인증 모듈 리팩터링 중
    [1] debugger — 세션 만료 오류 분석 중
    [2] explorer — 의존성 트리 탐색 중
```

스레드 전환 시 이전 스레드의 작업은 백그라운드에서 계속 진행됩니다. `/status` 로 모든 스레드의 현황을 확인할 수 있습니다.

## fork로 가설 분기

디버깅 중 두 가지 접근법을 동시에 시도하고 싶을 때 `codex fork` 를 사용합니다. 현재 대화 상태를 복사해 새 세션을 만들므로, 각 세션에서 독립적으로 가설을 검증할 수 있습니다.

```bash
# 현재 세션에서 가설 A 검증 중
# fork로 동일 상태의 새 세션 생성
codex fork

# 원래 세션: 가설 A 계속
# 새 세션: 가설 B 시도

# 세션 목록 확인 후 결과가 좋은 세션으로 복귀
codex resume --all
codex resume <SESSION_ID>
```

<ConceptCard title="빌트인 에이전트 역할">

Codex에는 세 가지 빌트인 에이전트 역할이 있습니다.

- **default**: 일반 작업. 사용자와 직접 대화
- **worker**: 독립 태스크 실행. 사용자 입력 없이 작업 완료
- **explorer**: 읽기 전용 탐색. 코드 구조 파악, 근본 원인 분석에 적합

디버깅 시나리오에서는 메인 스레드를 `default`, 원인 분석 서브에이전트를 `explorer` 역할로 구성하는 패턴이 효과적입니다.

</ConceptCard>

## 실제 디버깅 시나리오

세션 만료 오류가 간헐적으로 발생하는 상황을 예시로 살펴봅니다.

```bash
# 1. 읽기 전용으로 탐색 에이전트 시작
codex -s read-only "auth/session.ts의 만료 로직을 분석하고 경쟁 조건 가능성을 찾아줘"

# 2. 분석 결과를 바탕으로 fork해서 두 가지 수정 방향 동시 시도
codex fork

# 3. 각 세션에서 다른 수정 적용 후 테스트
# 세션 A: mutex 추가
# 세션 B: 만료 시간 로직 재작성

# 4. 더 나은 결과 세션으로 돌아가 마무리
codex resume <better-session-id>
```

:::tip[팁]
`codex resume --all` 은 모든 저장된 세션 목록을 표시합니다. 세션 ID를 기억하지 못해도 내용을 보고 선택할 수 있습니다.
:::

## 핵심 정리

- **`spawn_agent`** 로 서브에이전트를 생성하고 병렬로 디버깅 가설을 검증할 수 있습니다
- **`/agent`** 슬래시로 TUI에서 실행 중인 에이전트 스레드를 즉시 전환합니다
- **`codex fork`** 는 현재 대화 상태를 복사해 독립적인 분기 세션을 만듭니다
- **`codex resume <SESSION_ID>`** 로 원하는 세션으로 언제든 돌아올 수 있습니다
- 읽기 전용 `explorer` 역할 에이전트는 코드 탐색과 근본 원인 분석에 최적입니다
- 에이전트 한도 (`max_threads`, `max_depth`) 는 config.toml 의 `[agents]` 섹션에서 조정합니다
