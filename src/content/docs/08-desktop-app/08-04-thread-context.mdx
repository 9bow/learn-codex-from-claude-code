## 컨텍스트 유실 문제

터미널에서 Codex CLI를 사용할 때 가장 번거로운 점 중 하나는 여러 프로젝트를 오가며 작업할 때 컨텍스트가 흩어진다는 것입니다. 프로젝트 A 작업을 잠시 멈추고 프로젝트 B로 전환하면, 다시 A로 돌아올 때 어디까지 했는지 파악하는 데 시간이 걸립니다.

Codex Desktop App은 **프로젝트별 스레드** 구조로 이 문제를 해결합니다. 각 프로젝트의 대화 기록과 상태가 독립적으로 유지됩니다.

## 스레드 구조

Desktop App에서 스레드는 다음과 같이 구성됩니다.

```
프로젝트: learn-codex
  ├── 스레드: 섹션 07 챕터 작성
  │     └── 대화 기록, 열린 파일, 마지막 diff
  ├── 스레드: 빌드 오류 수정
  │     └── 대화 기록, 오류 로그, 수정 시도 이력
  └── 스레드: 컴포넌트 리팩터링
        └── 대화 기록, 변경 파일 목록

프로젝트: api-server
  ├── 스레드: 인증 모듈 개선
  └── 스레드: 성능 프로파일링
```

각 스레드는 **독립적인 컨텍스트** 를 유지합니다. 스레드를 전환해도 이전 스레드의 대화와 상태는 그대로 보존됩니다.

## 동시 다중 스레드

Desktop App에서는 여러 스레드를 동시에 활성화할 수 있습니다. 스레드 A가 실행 중인 동안 스레드 B로 전환해 작업하고, 다시 A로 돌아오면 A의 실행 결과가 기다리고 있습니다.

:::tip[팁]
시간이 오래 걸리는 작업(대규모 리팩터링, 테스트 생성 등)은 별도 스레드로 시작하고, 다른 스레드에서 빠른 작업을 처리하는 방식으로 대기 시간을 줄일 수 있습니다.
:::

## diff 검토 워크플로

에이전트가 파일을 수정하면 Desktop App의 diff 검토 화면에서 다음 작업을 할 수 있습니다.

| 액션 | 설명 |
|------|------|
| **코멘트 달기** | 특정 줄에 코멘트를 달고 에이전트에게 수정 요청 |
| **에디터에서 열기** | VS Code 등 연결된 에디터에서 변경 파일 즉시 열기 |
| **부분 승인** | 변경 중 일부만 선택해서 적용 |
| **전체 승인** | 에이전트의 모든 변경을 로컬에 적용 |
| **거부** | 해당 스레드의 변경을 폐기 |

```
diff 검토 화면 예시:

  auth/session.ts  +42 -8
  ─────────────────────────────
  - const timeout = 3600;
  + const timeout = SESSION_TIMEOUT_SECONDS;   [💬 코멘트: 상수 위치가 어디인가요?]

  [에디터에서 열기]  [부분 승인]  [전체 승인]  [거부]
```

:::note[참고]
"에디터에서 열기" 기능은 Desktop App이 시스템에 설치된 기본 에디터를 감지합니다. VS Code가 설치되어 있으면 `code` 명령으로 해당 파일을 열고 변경 줄로 포커스를 이동합니다.
:::

<ConceptCard title="스레드 컨텍스트와 CLI 세션의 관계">

Desktop App의 스레드는 내부적으로 CLI 세션(`~/.codex/sessions/`)과 연결됩니다. Desktop App에서 시작한 스레드를 CLI에서 `codex resume <SESSION_ID>` 로 이어받을 수 있고, 반대로 CLI 세션을 Desktop App에서 스레드로 열 수도 있습니다.

이 덕분에 Desktop App에서 작업을 시작하고 서버에서 `codex resume` 으로 이어받는 식의 크로스 서피스 워크플로가 가능합니다.

</ConceptCard>

## 핵심 정리

- Desktop App의 **프로젝트별 스레드** 구조는 여러 프로젝트를 오가며 작업할 때 컨텍스트를 유실 없이 보존합니다
- 여러 스레드를 **동시에 활성화** 해 시간이 오래 걸리는 작업과 빠른 작업을 병행할 수 있습니다
- **diff 검토 화면** 에서 코멘트 달기, 에디터 열기, 부분 승인이 모두 가능합니다
- Desktop App 스레드는 CLI 세션과 연결되어 **크로스 서피스 재개** 가 가능합니다
- 스레드 전환 시 이전 스레드의 대화·상태·diff 가 **즉시 복원** 됩니다
- 코멘트를 달면 에이전트가 해당 피드백을 반영해 수정을 이어갑니다
