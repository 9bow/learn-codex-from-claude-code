## Automations란

Automations는 Codex Desktop App에서 제공하는 스케줄 기반 백그라운드 자동 작업 기능입니다. 정해진 일정에 따라 에이전트가 자동으로 실행되고, 결과는 Triage inbox에 쌓입니다. 개발자는 원하는 시간에 대기열을 열어 결과를 확인하고 적용 여부를 결정합니다.

Claude Code의 `/loop` 나 `CronCreate` 와 유사한 역할이지만, Desktop App의 시각적 관리 인터페이스와 통합되어 있습니다.

## Automations 구성 요소

Automation 하나는 다음 세 가지 요소로 구성됩니다.

| 요소 | 역할 | 예시 |
|------|------|------|
| **스케줄** | 실행 주기 | 매일 오전 9시, 매시간, cron syntax |
| **지침** | 에이전트에게 전달할 작업 내용 | "미해결 이슈를 분류하고 레이블을 붙여 줘" |
| **스킬** | 작업에 필요한 외부 도구 | `$linear`, `$github`, `$slack` |

```
Automation 예시:
  스케줄:  매일 오전 8시
  스킬:    $github
  지침:    어제 실패한 CI 워크플로를 요약하고
           원인별로 분류해 줘.
           같은 원인이 반복되면 원인 분석 이슈를 생성해 줘.
```

## 활용 사례

공식 Automations 문서는 반복 점검, 장기 실행 작업 확인, PR 상태 확인, 최근 커밋 요약 같은 recurring task에 Automations를 사용할 수 있다고 안내합니다.

## 검토 대기열 워크플로

Automation이 실행될 때마다 결과가 Desktop App의 Triage inbox에 추가됩니다. 개발자는 대기열에서 각 결과를 확인하고 다음 중 하나를 선택합니다.

- **승인**: 에이전트의 변경·액션을 적용합니다
- **수정**: 에이전트 결과를 기반으로 추가 지시를 내립니다
- **거부**: 이번 실행 결과를 무시합니다

<ConceptCard title="스킬과의 조합">

Automations는 Skills와 함께 사용할 때 가장 강력합니다. 예를 들어 `$linear` 스킬이 설치되어 있으면 다음과 같이 설정할 수 있습니다.

```
스케줄:  매주 월요일 오전 9시
스킬:    $linear, $github
지침:    지난 주 완료된 이슈와 PR을 매핑하여
         Linear의 릴리스 항목을 업데이트해 줘.
         연결되지 않은 PR이 있으면 목록으로 보고해 줘.
```

Automations는 Codex에서 사용할 수 있는 plugins와 skills를 함께 사용할 수 있습니다. 팀에서 반복 작업을 공유하려면 skill로 작업 절차를 먼저 정의하는 편이 유지보수에 유리합니다.

</ConceptCard>

:::tip[팁]
Automation 지침에 결과 형식을 명시하면 검토가 훨씬 편해집니다. "마크다운 표 형식으로 요약해 줘", "심각도별로 정렬해 줘" 와 같은 지시를 포함하세요.
:::

## 핵심 정리

- **Automations** 는 스케줄 + 지침 + 스킬을 조합한 Desktop App의 백그라운드 자동 작업 기능입니다
- 실행 결과는 **검토 대기열** 에 누적되며 개발자가 원하는 시간에 검토·승인합니다
- Claude Code의 `/loop`·`CronCreate` 와 유사하지만 Desktop App UI와 통합되어 있습니다
- **Skills** 와 조합하면 GitHub·Linear·Slack 등 외부 도구를 활용한 복잡한 자동화가 가능합니다
