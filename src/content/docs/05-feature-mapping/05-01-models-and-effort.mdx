## Claude Code의 모델 체계

Claude Code는 Anthropic의 세 가지 모델 계층을 제공합니다. 대화 중 `/model` 슬래시 커맨드로 전환하거나, `/effort` 로 추론 깊이를 조정할 수 있습니다.

| 모델 | 특징 | 적합한 용도 |
|------|------|-------------|
| `claude-opus-4-6` | 최고 성능, 느림 | 복잡한 설계, 심층 분석 |
| `claude-sonnet-4-6` | 균형, 기본값 | 일반 코딩, 리팩터링 |
| `claude-haiku-4-5` | 빠르고 저렴 | 빠른 수정, 서브에이전트 |

`/effort` 커맨드는 `low / medium / high` 세 단계로 추론 깊이를 조절합니다. 예를 들어 `/effort low` 는 빠른 응답이 필요한 단순 작업에 사용합니다.

## Codex의 모델 라인업

Codex는 GPT-5 계열 모델을 사용합니다. Claude Code보다 더 세분화된 라인업을 제공하며, 일부 모델은 특정 플랜에서만 사용할 수 있습니다.

| 모델 | 플래그 | 특징 | 비고 |
|------|--------|------|------|
| `gpt-5.5` | `-m gpt-5.5` | 복잡한 코딩, computer use, 지식 작업, 리서치에 권장 | ChatGPT 인증만, API 키 미지원 |
| `gpt-5.4` | `-m gpt-5.4` | 강한 reasoning, tool use, agentic workflow | API 키 지원 |
| `gpt-5.4-mini` | `-m gpt-5.4-mini` | 빠르고 효율적인 경량 코딩 모델 | API 키 지원, routine 작업·subagent에 적합 |
| `gpt-5.3-codex` | `-m gpt-5.3-codex` | 복잡한 소프트웨어 엔지니어링용 코딩 모델 | Codex Cloud에서도 사용 |
| `gpt-5.3-codex-spark` | `-m gpt-5.3-codex-spark` | near-instant coding iteration 연구 프리뷰 | ChatGPT Pro 사용자용, API 미지원 |
| `gpt-5.2` | `-m gpt-5.2` | 이전 범용 모델 | 대체 모델 |

:::note[참고]
`gpt-5.5` 는 2026-04-23 정식 출시되어 2026-05 기준 권장 기본 모델입니다. 단, ChatGPT 인증(Pro/Plus 플랜) 사용자만 이용 가능하며, API 키만 보유한 경우에는 `gpt-5.4` 가 권장 폴백 모델입니다.
:::

## model_reasoning_effort 설정

Codex에서 추론 효율은 `model_reasoning_effort` 설정으로 조정합니다. Claude Code의 `/effort` 커맨드와 동일한 역할이지만, 설정 파일에 영구 기록하거나 CLI 플래그로 실행 시점에 지정하는 방식입니다.

```toml
# ~/.codex/config.toml
model = "gpt-5.4"
model_reasoning_effort = "medium"   # minimal | low | medium | high | xhigh
```

또는 실행 시 직접 지정:

```bash
codex --model gpt-5.4 -c model_reasoning_effort='"high"' "이 함수를 리팩터링해줘"
```

| 값 | 특징 |
|----|------|
| `minimal` | 매우 낮은 latency가 중요한 단순 작업 |
| `low` | 빠른 응답, 단순 작업 |
| `medium` | 기본값, 균형 |
| `high` | 심층 추론, 복잡한 설계 |
| `xhigh` | 모델이 지원하는 경우 가장 깊은 추론 |

## 오픈소스 모델 사용

Codex CLI는 `--oss` 플래그로 Ollama나 LM Studio 같은 로컬 모델도 사용할 수 있습니다. Claude Code에는 없는 기능입니다.

```bash
codex --oss "로컬 모델로 코드 검토해줘"
```

AWS를 사용하는 팀은 `model_provider = "amazon-bedrock"` 을 config.toml에 추가하여 Bedrock을 통해 모델을 호출할 수 있습니다.

## 핵심 정리

- Claude Code의 `opus/sonnet/haiku` 는 Codex의 `gpt-5.5/gpt-5.4/gpt-5.4-mini` 에 대략 대응합니다
- **추론 효율** 조정은 Codex에서 `model_reasoning_effort` 설정을 사용합니다
- `gpt-5.5` 는 공식 Models 문서 기준 Codex의 권장 최신 모델이며 API 키 인증은 지원하지 않습니다
- `gpt-5.5` 는 ChatGPT 인증 사용자 전용이며, API 키 사용자에게는 `gpt-5.4` 를 권장합니다
- `gpt-5.4-mini` 는 빠르고 효율적인 경량 코딩 모델로 routine 작업과 subagent에 적합합니다
- `gpt-5.3-codex-spark` 는 Pro 플랜 전용으로, 빠른 반복 작업에 특화되어 있습니다
- Codex CLI는 `--oss` 플래그로 로컬 모델(Ollama, LM Studio)도 지원합니다
