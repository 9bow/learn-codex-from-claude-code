## Claude Code 슬래시 커맨드 목록

Claude Code TUI에서 `/` 를 입력하면 다음 커맨드를 사용할 수 있습니다.

| 커맨드 | 기능 |
|--------|------|
| `/help` | 사용 가능한 커맨드 목록 표시 |
| `/model` | 모델 전환 |
| `/effort` | 추론 깊이 조정 (low/medium/high) |
| `/clear` | 대화 내역 초기화 |
| `/resume` | 이전 세션 재개 |
| `/init` | 프로젝트 CLAUDE.md 초기화 |
| `/permissions` | 현재 권한 설정 확인 및 변경 |
| `/mcp` | MCP 서버 상태 확인 |
| `/memory` | 메모리 파일 편집 |
| `/compact` | 컨텍스트 압축 (요약으로 대체) |
| `/loop` | 반복 실행 모드 |

## Codex 슬래시 커맨드 목록

Codex TUI는 다음 슬래시 커맨드를 제공합니다.

| 커맨드 | 기능 |
|--------|------|
| `/model` | 모델 전환 |
| `/permissions` | 승인 정책 변경 (alias: `/approvals`) |
| `/theme` | UI 테마 전환 |
| `/review` | 리뷰어 서브에이전트 실행 |
| `/agent` | 에이전트 스레드 전환 |
| `/status` | 현재 세션 상태 표시 |
| `/copy` | 마지막 응답 클립보드 복사 (Ctrl+O) |
| `/clear` | 대화 내역 초기화 |
| `/feedback` | 피드백 제출 |
| `/exit` | 세션 종료 |
| `/personality` | 응답 스타일 전환 (concise / conversational) |

## 직접 매핑이 되는 커맨드

### `/model`

두 도구 모두 `/model` 로 모델을 전환합니다. 동작 방식은 거의 동일합니다.

### `/clear`

대화 내역을 초기화합니다. 두 도구 모두 동일한 커맨드입니다.

### `/permissions` / `/approvals`

Codex에서는 `/permissions` 와 `/approvals` 가 같은 커맨드입니다. Claude Code의 `/permissions` 와 개념적으로 동일하지만, Codex의 승인 정책(`approval_policy`)을 변경한다는 점에서 차이가 있습니다.

## 직접 매핑이 어려운 커맨드

### `/resume` → `codex resume`

Claude Code의 `/resume` 는 TUI 내에서 이전 세션을 재개합니다. Codex에서는 TUI 슬래시 커맨드가 아니라 CLI 명령어로 제공됩니다.

```bash
codex resume          # 세션 목록에서 선택
codex resume --last   # 마지막 세션 재개
```

### `/memory` → AGENTS.md 편집

Claude Code의 `/memory` 는 `~/.claude/CLAUDE.md` 를 에디터에서 열어줍니다. Codex에는 동일한 TUI 커맨드가 없으며, `~/.codex/AGENTS.md` 를 직접 편집해야 합니다.

### `/init` → 없음

Claude Code의 `/init` 은 현재 프로젝트를 분석하여 `CLAUDE.md` 를 자동 생성합니다. Codex에는 동일한 기능이 없으므로, `AGENTS.md` 를 수동으로 작성하거나 `$skill-creator` 스킬을 활용할 수 있습니다.

### `/compact` → 자동 관리

Claude Code의 `/compact` 는 컨텍스트가 길어졌을 때 요약으로 대체합니다. Codex는 컨텍스트 관리를 자동으로 처리하며, 별도 커맨드가 없습니다.

### `/effort` → `model_reasoning_effort`

TUI 슬래시 커맨드 대신 설정 파일에서 변경합니다.

```toml
model_reasoning_effort = "high"
```

<ConceptCard title="Codex 전용 커맨드">
Codex에만 있는 커맨드도 있습니다.

- `/review`: 리뷰어 서브에이전트를 즉시 실행합니다. Claude Code의 `code-reviewer` 에이전트와 유사하지만 TUI에서 바로 호출할 수 있습니다.
- `/agent`: 열려 있는 에이전트 스레드 간에 전환합니다. 멀티 에이전트 작업 시 유용합니다.
- `/personality`: 응답 스타일을 간결(concise) 또는 대화적(conversational)으로 전환합니다.
- `/theme`: UI 테마를 변경합니다.
</ConceptCard>

:::caution[주의]
Codex TUI에서 Tab 키는 커맨드 자동완성이 아닌 예약 키입니다. 커맨드 검색은 Ctrl+R 을 사용합니다.
:::

## 핵심 정리

- `/model` 과 `/clear` 는 두 도구에서 동일하게 동작합니다
- Claude Code의 `/permissions` 는 Codex의 `/permissions` (또는 `/approvals`) 에 대응합니다
- `/resume` 은 Codex에서 TUI 커맨드가 아닌 `codex resume` CLI 명령어로 제공됩니다
- `/init` 과 `/memory` 는 Codex에 직접 대응 커맨드가 없으며, AGENTS.md 직접 편집으로 대체합니다
- Codex 전용 커맨드인 `/review`, `/agent`, `/personality` 는 멀티 에이전트 워크플로를 지원하는 기능입니다
