## Profiles란

**Profiles** 는 `config.toml` 안에 이름 붙인 설정 블록을 미리 정의해 두고, 실행 시 `--profile` (또는 `-p`) 플래그 하나로 전체 설정을 바꿔 치는 기능입니다.

예를 들어 평소에는 빠른 `gpt-5.4-mini` 모델을 쓰다가, 깊은 코드 리뷰가 필요할 때만 `gpt-5.5` + `high` reasoning으로 전환하는 식입니다.

:::caution[실험적 기능]
Profiles는 **Experimental** 기능입니다. IDE 확장(VS Code, JetBrains)에서는 현재 지원되지 않으며, CLI 전용으로 동작합니다. 추후 변경될 수 있으니 공식 문서를 병행 확인하세요.
:::

## config.toml에서 프로파일 정의하기

`~/.codex/config.toml` (또는 프로젝트 `.codex/config.toml`)의 최상위에 기본 설정을 두고, `[profiles.<name>]` 섹션으로 오버라이드할 값만 선언합니다. 지정하지 않은 키는 최상위 값을 그대로 상속합니다.

```toml
# ~/.codex/config.toml

# 기본 설정
model = "gpt-5.4-mini"
model_reasoning_effort = "medium"
approval_policy = "on-request"
sandbox_mode = "workspace-write"

# 프로파일: 심층 리뷰용
[profiles.deep-review]
model = "gpt-5.5"
model_reasoning_effort = "high"
approval_policy = "on-request"
sandbox_mode = "read-only"

# 프로파일: 빠른 반복용
[profiles.lightweight]
model = "gpt-5.4-mini"
model_reasoning_effort = "low"
approval_policy = "never"
sandbox_mode = "workspace-write"
```

## 프로파일 전환하기

정의한 프로파일은 `--profile` (줄여서 `-p`) 플래그로 선택합니다.

```bash
# 심층 리뷰 모드
codex --profile deep-review "이 PR의 보안 취약점을 분석해 줘"

# 빠른 반복 모드
codex -p lightweight "주석 정리해 줘"

# 프로파일 없이 실행 (기본 설정 사용)
codex "테스트 실행해 줘"
```

:::tip[팁]
프로파일명은 소문자와 하이픈을 사용하는 것이 관례입니다. 탭 자동완성이 지원되는 셸에서는 `--profile ` 뒤에 Tab을 누르면 정의된 프로파일 목록이 나타납니다.
:::

## 실용적인 프로파일 시나리오

| 프로파일명 | 용도 | 권장 모델 | reasoning_effort |
|-----------|------|-----------|-----------------|
| `deep-review` | 보안·아키텍처 리뷰 | `gpt-5.5` | `high` |
| `lightweight` | 빠른 리팩터링, 주석 정리 | `gpt-5.4-mini` | `low` |
| `ci` | GitHub Actions 자동화 | `gpt-5.4` | `medium` |
| `interactive` | 대화형 페어 프로그래밍 | `gpt-5.3-codex` | `medium` |

## Claude Code와의 비교

Claude Code에는 Profiles에 직접 대응하는 내장 기능이 없습니다. Claude Code 사용자들은 보통 아래 두 가지 방법으로 비슷한 효과를 냈습니다.

<Compare>
  <Fragment slot="left">
  **Claude Code 대용 패턴**
  - `settings.local.json`에 일시 오버라이드 값을 수동으로 작성
  - `.claude/settings.json`을 여러 버전으로 관리 후 심볼릭 링크 교체
  - `/model` 슬래시 커맨드로 모델을 대화 중 전환
  - 쉘 alias로 `alias claude-deep='claude --model claude-opus-4-6'` 등록

  </Fragment>
  <Fragment slot="right">
  **Codex Profiles**
  - `config.toml` 한 곳에서 모든 프로파일 관리
  - `--profile <name>` 단일 플래그로 전환
  - 모델, reasoning effort, approval, sandbox 등 복합 설정을 한 번에 변경
  - 프로젝트 `.codex/config.toml`에 체크인해 팀 공유 가능

  </Fragment>
</Compare>

:::note[IDE 확장 사용자 주의]
VS Code / JetBrains Codex 확장에서는 Profiles가 아직 지원되지 않습니다. IDE에서 작업할 때는 `config.toml`의 최상위 값이 항상 사용됩니다. 프로파일이 필요한 작업은 터미널 CLI로 실행하세요.
:::

## 핵심 정리

- `[profiles.<name>]` 섹션으로 이름 붙인 설정 프리셋을 정의할 수 있습니다
- `codex --profile <name>` (또는 `-p`) 플래그로 즉시 전환합니다
- **Experimental** 기능으로 **IDE 확장에서는 미지원** 입니다
- 지정하지 않은 키는 최상위 설정을 자동 상속합니다
- Claude Code에는 직접 대응 기능이 없으며, `settings.local.json` 수동 편집이나 쉘 alias가 대용 패턴입니다
- 팀 공유가 필요하면 프로젝트 `.codex/config.toml`에 프로파일을 체크인하세요
