## Codex Cloud란

**Codex Cloud** 는 로컬 머신이 아닌 OpenAI 클라우드 인프라에서 에이전트 작업을 실행하는 서비스입니다. 사용자가 요청을 보내면 클라우드 샌드박스에서 코드를 읽고 수정한 뒤, 결과를 Pull Request로 제출하거나 Slack 메시지로 알려줍니다.

주요 특징을 정리하면 다음과 같습니다.

| 특징 | 설명 |
|------|------|
| 백그라운드 실행 | 로컬 터미널을 닫아도 작업 계속 진행 |
| GitHub 통합 | PR 자동 생성, 코드 리뷰 코멘트 작성 |
| Slack 통합 | 작업 완료 알림, Slack에서 직접 작업 지시 |
| 자동 코드 리뷰 | PR에 대한 자동 리뷰 코멘트 생성 |

:::caution[ChatGPT 인증 필요]
Codex Cloud는 API 키가 아닌 **ChatGPT 계정 인증** 을 사용합니다. `codex login` 으로 ChatGPT 계정에 로그인해야 하며, API Key만으로는 사용할 수 없습니다.
또한 Codex Cloud에서는 **모델 선택이 불가** 합니다. OpenAI가 내부적으로 최적 모델을 자동 선택합니다.
:::

## codex cloud 명령

```bash
# 클라우드 작업 목록 보기
codex cloud list

# 새 클라우드 작업 생성
codex cloud exec --env <ENV_ID> "이 저장소의 README를 업데이트하고 PR을 열어줘"
```

작업을 생성하려면 대상 Codex Cloud environment가 필요합니다. `codex cloud list` 로 최근 작업을 확인하고, 결과 diff를 로컬에 적용할 때 작업 ID를 사용합니다.

## codex apply — 결과를 로컬에 적용

클라우드에서 작업이 완료되면 변경사항을 로컬 워크스페이스에 직접 적용할 수 있습니다.

```bash
# 클라우드 작업 결과를 현재 디렉토리에 적용
codex apply <TASK_ID>
```

`codex apply` 는 클라우드가 만든 최신 diff를 현재 로컬 git 워크트리에 패치 형태로 적용합니다. 충돌이 있으면 non-zero로 종료합니다.

<ConceptCard title="GitHub에서 Codex Cloud 위임">
공식 Codex web 문서는 GitHub 계정을 연결한 뒤, 이슈나 Pull Request에서 `@codex`를 태그해 클라우드 작업을 시작하는 흐름을 안내합니다. 작업이 완료되면 Codex가 변경사항을 제안하고 PR로 이어갈 수 있습니다.
</ConceptCard>

## Claude Code와의 비교

  - GitHub 연동은 사용자가 직접 커밋/PR 수행
  - Slack 연동 없음 (별도 MCP 서버로 구성 가능)

  - Slack 워크스페이스 연동으로 알림 수신
  - 모델 선택 불가 (OpenAI 자동 선택)
  - ChatGPT 인증 필수 (API Key 불가)

## 핵심 정리

- Codex Cloud는 **클라우드 샌드박스** 에서 백그라운드로 에이전트 작업을 실행합니다
- GitHub PR 자동 생성, 코드 리뷰 코멘트, Slack 알림을 기본 제공합니다
- `codex cloud exec --env <ENV_ID> "..."` 로 작업을 생성하고, `codex apply <TASK_ID>` 로 결과를 로컬에 적용합니다
- **ChatGPT 계정 인증** 이 필수이며, API Key 방식은 지원하지 않습니다
- **모델 선택이 불가** 하며 OpenAI가 내부적으로 최적 모델을 선택합니다
- Claude Code에는 직접 대응하는 클라우드 백그라운드 실행 기능이 없습니다
