## 플러그인 시스템

Codex에서 **Skills** 는 재사용 가능한 작업 지침의 작성 형식이고, **plugins** 는 Skills, MCP 설정, app integration, lifecycle hooks 등을 묶어 설치·배포하는 단위입니다. 한 팀 안에서 반복 워크플로를 공유하려면 먼저 skill로 절차를 만들고, 여러 사람이 설치해야 할 시점에 plugin으로 패키징하는 흐름이 자연스럽습니다.

공식 문서 기준 plugin은 `.codex-plugin/plugin.json` manifest를 필수로 갖습니다. skill만 담는 최소 구조는 다음과 같습니다.

```text
my-first-plugin/
  .codex-plugin/
    plugin.json
  skills/
    hello/
      SKILL.md
```

```json
{
  "name": "my-first-plugin",
  "version": "1.0.0",
  "description": "Reusable greeting workflow",
  "skills": "./skills/"
}
```

## Marketplace 관리

Plugin marketplace는 Codex가 읽고 설치할 수 있는 plugin catalog입니다. repo 범위에서는 `$REPO_ROOT/.agents/plugins/marketplace.json`, 개인 범위에서는 `~/.agents/plugins/marketplace.json`에 둘 수 있습니다.

CLI로 marketplace source를 추가·갱신·제거할 수 있습니다.

```bash
codex plugin marketplace add owner/repo
codex plugin marketplace add owner/repo --ref main
codex plugin marketplace add https://github.com/example/plugins.git --sparse .agents/plugins
codex plugin marketplace add ./local-marketplace-root

codex plugin marketplace upgrade
codex plugin marketplace upgrade marketplace-name
codex plugin marketplace remove marketplace-name
```

<ConceptCard title="플러그인 vs Skills의 차이">
- **Skills**: 작업 절차 자체. `SKILL.md`에 `name`, `description`, 지침을 둡니다
- **Plugins**: 설치 가능한 배포 단위. Skills, MCP 서버, app integration, hooks를 함께 묶을 수 있습니다

작업 절차가 한 프로젝트 안에서만 필요하면 local skill로 충분합니다. 팀 배포, 앱 연동, MCP 설정까지 함께 전달해야 한다면 plugin이 더 적합합니다.
</ConceptCard>

## OpenAI Agents SDK와 Codex 연결

공식 가이드는 별도의 `@openai/codex-sdk` 패키지를 쓰는 방식이 아니라, **Codex CLI를 MCP 서버로 실행하고 OpenAI Agents SDK에서 MCP tool로 호출하는 방식** 을 안내합니다.

```bash
codex mcp-server
```

이 MCP 서버는 새 Codex 대화를 시작하는 `codex` tool과 기존 thread를 이어가는 `codex-reply` tool을 노출합니다. Agents SDK 쪽에서는 `MCPServerStdio`로 이 서버를 띄우고, 전문 agent들이 필요한 시점에 Codex tool을 호출하도록 구성합니다.

```python
from agents import Agent, Runner
from agents.mcp import MCPServerStdio

async with MCPServerStdio(
    name="Codex CLI",
    params={"command": "npx", "args": ["-y", "codex", "mcp-server"]},
) as codex_mcp_server:
    developer = Agent(
        name="Developer",
        instructions="Use Codex for scoped code changes.",
        mcp_servers=[codex_mcp_server],
    )

    await Runner.run(developer, "Add a focused regression test.")
```

이 방식의 장점은 orchestration, handoff, guardrails, trace를 Agents SDK가 담당하고, 실제 코드 작업은 Codex CLI의 sandbox·approval·AGENTS.md 컨텍스트를 활용한다는 점입니다.

## GitHub 자동화는 어떻게 보나

OpenAI 공식 Codex 문서에서 확인되는 경로는 GitHub Action 패키지(`openai/codex-action@v1`)가 아니라 **Codex Cloud와 GitHub 통합** 입니다. PR·이슈 기반 작업은 Cloud 환경을 연결한 뒤 `@codex` 위임이나 Cloud task로 처리하는 흐름을 우선 검토하세요.

## 핵심 정리

- **Skills** 는 작업 절차, **plugins** 는 설치 가능한 배포 단위입니다
- Plugin은 `.codex-plugin/plugin.json` manifest를 필수로 갖고, skills/MCP/apps/hooks를 함께 묶을 수 있습니다
- Marketplace는 repo 또는 개인 범위의 plugin catalog이며 `codex plugin marketplace add|upgrade|remove`로 관리할 수 있습니다
- SDK 기반 오케스트레이션은 Codex CLI를 MCP 서버로 노출하고 OpenAI Agents SDK에서 호출하는 방식이 공식 가이드에 정리되어 있습니다
- 공식 문서에서 확인되지 않는 `openai/codex-action@v1`, `@openai/codex-sdk`, `Code Mode`, `JS REPL Runtime` 예시는 사용하지 않는 편이 안전합니다
