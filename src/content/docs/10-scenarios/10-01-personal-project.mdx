## Day 1 마이그레이션 흐름

개인 사이드 프로젝트의 마이그레이션은 네 단계로 진행합니다.

```
1. Codex CLI 설치 및 인증
       ↓
2. AGENTS.md 작성 (CLAUDE.md 참고)
       ↓
3. config.toml 변환 (settings.json 참고)
       ↓
4. 첫 codex 실행 및 확인
```

## 1단계: 설치 및 인증 확인

이미 설치되어 있다면 버전을 확인합니다.

```bash
codex --version
# codex 0.x.x

# 인증 상태 확인
codex login status
# Logged in as: your@email.com
```

미설치 상태라면 [03-install](/learn-codex-from-claude-code/03-install/) 섹션을 먼저 완료하세요.

## 2단계: AGENTS.md 작성

프로젝트 루트에 `AGENTS.md` 파일을 만듭니다. 기존 `CLAUDE.md` 내용을 그대로 복사하면 대부분 작동하지만, 아래 항목들은 반드시 확인하고 수정해야 합니다.

<Compare>
  <Fragment slot="left">
  **CLAUDE.md에서 주의할 내용**
  - Claude Code 전용 슬래시 커맨드 언급 (`/model`, `/effort` 등)
  - `settings.local.json` 참조
  - Claude Code hooks 설명 (`PreToolUse` 이벤트명 등)
  - `claude-opus-4-6` 같은 Claude 모델명 직접 언급

  </Fragment>
  <Fragment slot="right">
  **AGENTS.md로 변환 시**
  - Codex 슬래시 커맨드로 대체하거나 삭제
  - `config.toml` 참조로 변경
  - Codex hooks 문법으로 변경 (또는 삭제 후 hooks.json 별도 관리)
  - Codex 모델명(`gpt-5.4` 등)으로 대체 또는 모델 언급 제거

  </Fragment>
</Compare>

기본 `AGENTS.md` 템플릿입니다.

```markdown
# 프로젝트 개요
[프로젝트 이름]은 [한 줄 설명]입니다.

## 기술 스택
- 언어: TypeScript
- 런타임: Node.js 22
- 패키지 매니저: pnpm

## 코드 스타일
- 함수는 50줄 이내로 유지
- 불변성(immutability) 패턴 사용
- 에러는 반드시 핸들링

## 작업 지침
- 변경 전 반드시 테스트 실행: `pnpm test`
- 빌드 확인: `pnpm build`
- 커밋 전 린트: `pnpm lint`

## 금지 사항
- `node_modules/` 직접 편집 금지
- `.env` 파일 수정 금지
```

:::tip[project_doc_fallback_filenames 활용]
기존 `CLAUDE.md`를 그대로 유지하면서 Codex에서도 읽히게 하려면 `config.toml`에 다음을 추가합니다.

```toml
project_doc_fallback_filenames = ["CLAUDE.md"]
```

이렇게 하면 `AGENTS.md`가 없을 때 `CLAUDE.md`를 폴백으로 사용합니다. 두 도구를 병행하는 동안 임시 방편으로 유용합니다.
:::

## 3단계: config.toml 변환

기존 `.claude/settings.json`을 참고해 `.codex/config.toml`을 만듭니다.

```toml
# .codex/config.toml

# 모델 설정
model = "gpt-5.4"
model_reasoning_effort = "medium"

# 승인 정책
approval_policy = "on-request"
sandbox_mode = "workspace-write"

# 웹 검색
web_search = "cached"

# 히스토리
[history]
persistence = "save-all"
```

`settings.json`의 `permissions.allow` 목록에 해당하는 내용은 `approval_policy`와 `sandbox_mode`로 표현합니다. 세밀한 제어가 필요하면 `approval_policy = { granular = { ... } }` 형식과 Rules를 함께 활용합니다.

## 4단계: 첫 실행

```bash
cd ~/my-side-project

# 기본 실행
codex

# 또는 작업 지시와 함께 바로 시작
codex "현재 코드에서 TypeScript 오류를 모두 찾아 수정해 줘"
```

처음 실행 시 **project trust** 프롬프트가 나타납니다. `y` 를 입력하면 프로젝트 `.codex/config.toml`과 `AGENTS.md`가 로드됩니다.

```
This project contains a .codex/config.toml file.
Do you want to trust and load it? [y/N]
```

<ConceptCard title="base path 자동 감지">
`config.toml`에 `base_path`를 별도로 설정하지 않으면 Codex가 `git rev-parse --show-toplevel` 로 자동 감지합니다. 모노레포처럼 특정 하위 디렉토리를 루트로 지정해야 할 때는 명시적으로 설정합니다.

```toml
# .codex/config.toml
base_path = "./packages/my-app"
```
</ConceptCard>

## 핵심 정리

- Day 1 마이그레이션은 **설치 확인 → AGENTS.md → config.toml → 첫 실행** 순서로 진행합니다
- `CLAUDE.md`를 직접 복사할 때는 Claude 전용 슬래시 커맨드, 모델명, hooks 문법을 반드시 수정하세요
- `project_doc_fallback_filenames = ["CLAUDE.md"]` 로 두 도구를 병행하는 동안 임시 폴백을 설정할 수 있습니다
- 첫 실행 시 **project trust** 프롬프트에 `y` 를 입력해야 프로젝트 설정이 로드됩니다
- `base_path`는 git 루트를 자동 감지하며, 모노레포 환경에서만 수동 지정이 필요합니다
